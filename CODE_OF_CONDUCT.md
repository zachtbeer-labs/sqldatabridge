# Code of Conduct

This is a small project with a simple ask: be good to each other and be good to the code.

## The Rules

1. **Be a good steward of the code.** Leave things better than you found them. If you break something, fix it. If you see something broken, say something.

2. **Don't spam.** No drive-by self-promotion, no bot-generated issues, no AI-generated PRs without actually reading what you're submitting.

3. **Write tests.** If you're adding or changing behavior, prove it works. Untested code is a gift that keeps on taking.

4. **Be the person your dog thinks you are.** Kind, enthusiastic, and genuinely happy to see others succeed. If you don't have a dog, be the person your mother wanted you to be.

## That's It

No 4,000-word legalese document needed. If you're the kind of person who reads a Code of Conduct before contributing to an open source project, you're almost certainly already following these rules.
